Prerequisites

Install the required compilation tools and libraries for your development machine.

  • Xcode Command Line Tools: install via terminal: 
    xcode-select --install
  • CMake: install via Homebrew: 
    brew install cmake ninja
  • Visual Studio 2022 (Community or higher): Make sure the "Desktop development with C++" workload is selected during installation.
  • CMake: install using the Windows installer from cmake.org, or package manager: 
    winget install Kitware.CMake
  • Git: ensure Git for Windows is installed and added to your system PATH.
  • Compiler & Build Tools: Atmospheric requires a C++17 compiler and basic windowing libraries: 
    # Ubuntu / Debian
sudo apt update && sudo apt install build-essential cmake ninja-build libx11-dev libxcursor-dev libxinerama-dev libxrandr-dev libxi-dev libgl1-mesa-dev pkg-config

# Arch Linux
sudo pacman -S base-devel cmake ninja xorg-server mesa

Repository Setup

Clone Atmospheric-Starter, a cross-platform starter app built on Atmospheric — it pulls in the engine as a submodule, so this is the fastest way to get a real build running.

# Clone the starter with nested submodules (Atmospheric + its own vcpkg submodule)
git clone --recurse-submodules https://github.com/painfulexistence/Atmospheric-Starter.git
cd Atmospheric-Starter
# Bootstrap vcpkg dependency manager
./Atmospheric/vcpkg/bootstrap-vcpkg.sh -disableMetrics
# Bootstrap vcpkg dependency manager (PowerShell / cmd)
.\Atmospheric\vcpkg\bootstrap-vcpkg.bat -disableMetrics

Build for Target Platform

Select your compilation target platform to view the build instructions.

Configure and build using the bundled CMake presets — the same commands work on macOS, Windows, and Linux.

cmake --preset desktop
cmake --build --preset debug   # or --preset release

The first configure bootstraps vcpkg and builds all dependencies from source, so it takes a while — subsequent builds are fast.

# Run the example binary
./build/GradientQuad/Debug/GradientQuad
# Run the example binary
.\build\GradientQuad\Debug\GradientQuad.exe

Compile GradientQuad to WebAssembly using Emscripten.

Note: Ensure you have the Emscripten SDK (emsdk) activated in your shell (e.g. source path/to/emsdk/emsdk_env.sh) before configuring.

cmake --preset wasm
cmake --build --preset wasm

Serve and open the build output in a browser:

python -m http.server 8000 --directory build-wasm/GradientQuad
# then open http://localhost:8000/GradientQuad.html

Build the Android APK. Shaders, assets, and native dependencies are packed automatically by Gradle, which drives the same CMake build under the hood.

Prerequisite: Ensure ANDROID_HOME points to your Android SDK installation. The required NDK version is pinned in platforms/android/app/build.gradle.kts and is downloaded automatically on first build.

cd platforms/android
./gradlew assembleDebug

# Output APK path:
# app/build/outputs/apk/debug/app-debug.apk
cd platforms\android
.\gradlew.bat assembleDebug

# Output APK path:
# app\build\outputs\apk\debug\app-debug.apk

Configure and open the Xcode project to run GradientQuad on the iOS Simulator or a device.

# Generate the Xcode project for the iOS Simulator
cmake -G Xcode -B build-ios-sim -S platforms/ios \
  -DCMAKE_SYSTEM_NAME=iOS \
  -DCMAKE_OSX_SYSROOT=iphonesimulator \
  -DCMAKE_OSX_DEPLOYMENT_TARGET=16.0 \
  -DVCPKG_TARGET_TRIPLET=arm64-ios-simulator

# Open in Xcode, or build from the CLI:
cmake --build build-ios-sim --config Release

For a physical device, swap the triplet/sysroot flags for arm64-ios / iphoneos and pass -DCMAKE_XCODE_ATTRIBUTE_DEVELOPMENT_TEAM=<TeamID> to configure code signing.

iOS Build Unavailable: Building iOS applications is only supported from a macOS host machine with Xcode installed.
